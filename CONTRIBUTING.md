# Contributing to Awesome Long-Horizon Agents 🌱

Thanks for helping this list grow! This reading list accompanies the survey
**_Toward Long-Horizon AI Agents_**, and it's meant to be a *living* resource.
Whether you're adding a paper we missed, fixing a broken link, or proposing a
new sub-topic — **your PR is welcome.**

## 📌 What belongs here

Papers, systems, benchmarks, and resources about **long-horizon agents** — agents
that plan and act over extended horizons in real environments, recover from
mistakes, and adapt during execution. Use the six-perspective taxonomy in the
[README](README.md) to place your entry:

**Foundations · Evolution · Harnesses · Optimization · Applications · Frontiers**

If an entry could fit multiple sections, pick the *primary* one and add it once.

## ✍️ Entry format

Keep every entry consistent with its neighbors — a bold title, a one-line takeaway,
then a metadata row of badges (venue · arXiv/Website · live GitHub stars):

```markdown
- **Paper Title** — one-line takeaway.<br/>![venue](https://img.shields.io/badge/Venue%20Year-555?style=flat-square) [![arXiv](https://img.shields.io/badge/arXiv-XXXX.XXXXX-b31b1b?style=flat-square&logo=arxiv&logoColor=white)](https://arxiv.org/abs/XXXX.XXXXX) [![stars](https://img.shields.io/github/stars/OWNER/REPO?style=flat-square&logo=github&label=%E2%98%85&color=444)](https://github.com/OWNER/REPO)
```

- The **stars badge is live** — it pulls the current count from GitHub automatically; just fill in `OWNER/REPO`.
- No arXiv? Use a Website/Docs badge instead: `[![site](https://img.shields.io/badge/%F0%9F%8C%90_Website-0E9B9B?style=flat-square)](URL)`.

Guidelines:

- **Link to the arXiv `abs/` page** (not the PDF) in the arXiv badge when available;
  otherwise the official page or the ACL Anthology in a Website badge.
- Add the **stars badge** when a repo exists; drop it when there's no code. Omit what doesn't exist.
- The takeaway is **one line** — say what's distinctive, not the whole abstract.
- Mark clear milestones or highly-cited work with a ⭐ before the title.
- Prefer **representative** entries: a few strong papers per category is the goal.
  This is a curated list, not an exhaustive dump.
- Keep chronological-ish or importance ordering within a subsection; don't worry
  about being exact.

## 🚀 How to contribute

1. **Fork** the repo and create a branch.
2. Add your entry (or entries) in the right section of `README.md`.
3. Double-check the link renders and points to the right paper.
4. Open a **Pull Request** with a short description (title + which section).

For anything larger — a new sub-topic, a restructure, or a batch of papers —
please **open an Issue first** so we can discuss placement.

## ✅ Checklist before opening a PR

- [ ] Entry is in the correct section and matches the format above.
- [ ] Link works and points to the canonical source.
- [ ] No duplicate of an existing entry.
- [ ] One-line takeaway, no marketing fluff.
